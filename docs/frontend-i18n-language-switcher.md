# Frontend i18n Language Switcher

## Overview
This document describes the lightweight frontend internationalization (i18n) setup for the SocialListening dashboard. The architecture is designed to be minimal, safe, and easily extensible without requiring heavy third-party libraries or altering backend APIs.

## Supported Languages
- `vi`: Vietnamese (Tiếng Việt) - **Default**
- `en`: English
- `th`: Thai (ภาษาไทย)
- `ja`: Japanese (日本語)
- `ko`: Korean (한국어)
- `zh`: Chinese (中文)

## Architecture

1. **Context & Provider**: 
   - `LanguageProvider` (`frontend/src/contexts/LanguageContext.tsx`) manages the current language state.
   - It exposes the `t(key)` function to retrieve translated text and `setLanguage(lang)` to change the locale.
2. **Storage**:
   - The selected language is stored in `localStorage` under the key `app_language`.
   - On initial load, the provider safely hydrates this value to persist the user's preference across reloads.
3. **Dictionaries**:
   - Stored as simple TypeScript objects in `frontend/src/i18n/locales/*.ts`.
4. **UI Switcher**:
   - `LanguageSwitcher` (`frontend/src/components/LanguageSwitcher.tsx`) is a dropdown component added to the top dashboard header.

## How to add a new translation key

1. Open `frontend/src/i18n/locales/vi.ts`.
2. Add your new key in the appropriate category (or create a new category):
   ```typescript
   export const vi = {
     nav: {
       ...
       newFeature: 'Tính năng mới',
     }
   }
   ```
3. Replicate this key across all other locale files (`en.ts`, `th.ts`, etc.) to ensure translations exist.
4. In your component, use the `t` function:
   ```typescript
   const { t } = useLanguage();
   <p>{t('nav.newFeature')}</p>
   ```

## How to add a new language

1. Create a new locale file, e.g., `frontend/src/i18n/locales/es.ts` (for Spanish).
2. Export the translation object mapping all keys identically to `vi.ts`.
3. Open `frontend/src/i18n/index.ts`.
4. Import your new locale and add it to the `dictionaries` object.
5. Add the language's display name to the `languageNames` object:
   ```typescript
   export const languageNames: Record<Language, string> = {
     ...
     es: 'Español',
   };
   ```

## Intentionally Not Translated
The following items are kept in their original language by design, to maintain data integrity and reduce complexity in this iteration:
- **User-generated content**: Project names, keywords, saved filters.
- **Crawled mentions**: The actual posts, comments, titles, and snippets scraped from external sources.
- **AI Content**: Summaries and analysis generated by the AI (though prompts can be localized in the future, the current return payloads remain untouched).
- **Backend errors/alerts**: Core backend schemas and Python validation errors remain in their original configurations.
