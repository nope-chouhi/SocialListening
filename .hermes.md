# Hermes Rules for Nope / SocialListening

When working in this repository, Hermes must follow these project rules.

## Mandatory workflow

1. Read AGENTS.md, .agents/rules/*.md, .cursorrules, and docs/hermes-telegram-antigravity-workflow.md before starting.
2. Pull latest main.
3. Create a separate branch for every task.
4. Implement the requested fix or feature.
5. Add or update an appropriate .md documentation file.
6. Run relevant tests, type-check, and build checks.
7. Commit changes to the branch.
8. Push the branch.
9. Stop and ask the user for confirmation before creating a PR.
10. Only create PR and merge after the user explicitly confirms.
11. Never deploy unless the user explicitly requests deployment.

## Forbidden

- Do not commit directly to main.
- Do not push directly to main.
- Do not force push main.
- Do not auto-merge without user confirmation.
- Do not run deploy.bat, production migrations, production restarts, or manual deploy scripts unless explicitly requested.
- Do not create fake UI, fake API, fake exports, fake buttons, dummy data, or mock production behavior.
- Do not delete untracked/newly installed files unless the user explicitly confirms deletion.

## Product philosophy

Data Quality > Data Quantity  
Trust > Fancy AI  
Source Coherence > UI Beauty  
Correctness > Demo Speed

## Completion report format

At the end of every task, report:

- Branch name
- Files changed
- Docs updated
- Tests/checks run
- Result
- Remaining risks
- Ask: "Bạn có muốn tôi tạo PR và merge sau khi checks pass không?"
